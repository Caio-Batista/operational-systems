<h4>Conclusões e expectativas:</h4>
<p>Uma das conclusões que podemos retirar após analisar os gráficos, é que nem sempre o algoritmo mais simples é o menos eficiente e o algoritmo mais complexo é o mais eficiênte (com menos page faults). 
Esperávamos que o algoritmo aging fosse melhor, pois teoricamente é o que é implementado no kernel do linux, porém teve casos que o second-chance foi melhor.</p>
